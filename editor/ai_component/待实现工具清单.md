# AI Component 待实现工具清单

## 概述

本文档列出了AI Component接下来需要实现的所有工具，涵盖了游戏开发过程中的各种操作。这些工具将允许AI模型与Godot引擎进行深度交互，执行从基本文件操作到复杂场景编辑的各种任务。

## 工具分类

### 1. 文件系统操作工具

#### 1.1 文件操作
- `create_file`: 创建新文件
  - 参数: path (文件路径), content (文件内容，可选)
  - 功能: 在指定路径创建新文件，可选择性地写入初始内容
  
- `write_to_file`: 写入文件内容
  - 参数: path (文件路径), content (文件内容)
  - 功能: 将指定内容写入文件，覆盖原有内容
  
- `append_to_file`: 追加文件内容
  - 参数: path (文件路径), content (追加内容)
  - 功能: 在文件末尾追加内容
  
- `delete_file`: 删除文件
  - 参数: path (文件路径)
  - 功能: 删除指定文件
  
- `rename_file`: 重命名文件
  - 参数: old_path (原文件路径), new_path (新文件路径)
  - 功能: 将文件重命名并移动到新路径
  
- `copy_file`: 复制文件
  - 参数: source_path (源文件路径), destination_path (目标文件路径)
  - 功能: 将文件从源路径复制到目标路径

#### 1.2 目录操作
- `create_directory`: 创建目录
  - 参数: path (目录路径)
  - 功能: 创建新目录
  
- `delete_directory`: 删除目录
  - 参数: path (目录路径)
  - 功能: 删除指定目录及其所有内容
  
- `rename_directory`: 重命名目录
  - 参数: old_path (原目录路径), new_path (新目录路径)
  - 功能: 将目录重命名并移动到新路径
  
- `copy_directory`: 复制目录
  - 参数: source_path (源目录路径), destination_path (目标目录路径)
  - 功能: 将目录从源路径复制到目标路径

### 2. 场景操作工具

#### 2.1 场景节点操作
- `add_node`: 添加节点
  - 参数: scene_path (场景路径), parent_path (父节点路径), node_type (节点类型), node_name (节点名称)
  - 功能: 在指定场景的指定父节点下添加新节点
  
- `remove_node`: 删除节点
  - 参数: scene_path (场景路径), node_path (节点路径)
  - 功能: 从场景中删除指定节点
  
- `rename_node`: 重命名节点
  - 参数: scene_path (场景路径), node_path (节点路径), new_name (新名称)
  - 功能: 重命名场景中的指定节点
  
- `move_node`: 移动节点
  - 参数: scene_path (场景路径), node_path (节点路径), new_parent_path (新父节点路径)
  - 功能: 将节点移动到新的父节点下
  
- `duplicate_node`: 复制节点
  - 参数: scene_path (场景路径), node_path (节点路径), new_name (新节点名称，可选)
  - 功能: 复制指定节点

#### 2.2 节点属性操作
- `set_node_property`: 设置节点属性
  - 参数: scene_path (场景路径), node_path (节点路径), property_name (属性名称), property_value (属性值)
  - 功能: 设置节点的指定属性值
  
- `get_node_property`: 获取节点属性
  - 参数: scene_path (场景路径), node_path (节点路径), property_name (属性名称)
  - 功能: 获取节点的指定属性值
  
- `reset_node_property`: 重置节点属性
  - 参数: scene_path (场景路径), node_path (节点路径), property_name (属性名称)
  - 功能: 将节点的指定属性重置为默认值

#### 2.3 场景管理
- `create_scene`: 创建新场景
  - 参数: path (场景文件路径), root_node_type (根节点类型)
  - 功能: 创建包含指定根节点类型的新场景文件
  
- `load_scene`: 加载场景
  - 参数: path (场景文件路径)
  - 功能: 加载指定场景文件
  
- `save_scene`: 保存场景
  - 参数: scene_path (场景路径，可选)
  - 功能: 保存当前场景或指定场景
  
- `close_scene`: 关闭场景
  - 参数: scene_path (场景路径，可选)
  - 功能: 关闭当前场景或指定场景

### 3. 脚本操作工具

#### 3.1 脚本文件操作
- `create_script`: 创建脚本文件
  - 参数: path (脚本文件路径), script_type (脚本类型: GDScript, C#), template (模板名称，可选)
  - 功能: 创建新的脚本文件
  
- `delete_script`: 删除脚本文件
  - 参数: path (脚本文件路径)
  - 功能: 删除指定的脚本文件

#### 3.2 脚本绑定操作
- `attach_script`: 绑定脚本到节点
  - 参数: scene_path (场景路径), node_path (节点路径), script_path (脚本文件路径)
  - 功能: 将脚本文件绑定到指定节点
  
- `detach_script`: 解绑节点脚本
  - 参数: scene_path (场景路径), node_path (节点路径)
  - 功能: 解除节点与脚本的绑定

#### 3.3 脚本内容操作
- `add_script_function`: 添加脚本函数
  - 参数: script_path (脚本文件路径), function_name (函数名称), function_code (函数代码)
  - 功能: 在脚本中添加新函数
  
- `modify_script_function`: 修改脚本函数
  - 参数: script_path (脚本文件路径), function_name (函数名称), new_function_code (新函数代码)
  - 功能: 修改脚本中的现有函数
  
- `remove_script_function`: 删除脚本函数
  - 参数: script_path (脚本文件路径), function_name (函数名称)
  - 功能: 从脚本中删除指定函数

### 4. 信号操作工具

#### 4.1 信号连接操作
- `connect_signal`: 连接信号
  - 参数: scene_path (场景路径), source_node_path (源节点路径), signal_name (信号名称), target_node_path (目标节点路径), target_function_name (目标函数名称)
  - 功能: 连接源节点的信号到目标节点的函数
  
- `disconnect_signal`: 断开信号连接
  - 参数: scene_path (场景路径), source_node_path (源节点路径), signal_name (信号名称), target_node_path (目标节点路径), target_function_name (目标函数名称)
  - 功能: 断开源节点信号与目标节点函数的连接

#### 4.2 信号自定义操作
- `add_custom_signal`: 添加自定义信号
  - 参数: scene_path (场景路径), node_path (节点路径), signal_name (信号名称)
  - 功能: 为节点添加自定义信号
  
- `remove_custom_signal`: 删除自定义信号
  - 参数: scene_path (场景路径), node_path (节点路径), signal_name (信号名称)
  - 功能: 删除节点的自定义信号

### 5. 资源操作工具

#### 5.1 资源导入/导出
- `import_resource`: 导入资源
  - 参数: source_path (源文件路径), target_path (目标资源路径)
  - 功能: 将外部文件导入为Godot资源
  
- `export_resource`: 导出资源
  - 参数: resource_path (资源路径), export_path (导出文件路径)
  - 功能: 将Godot资源导出为外部文件

#### 5.2 资源属性操作
- `set_resource_property`: 设置资源属性
  - 参数: resource_path (资源路径), property_name (属性名称), property_value (属性值)
  - 功能: 设置资源的指定属性值
  
- `get_resource_property`: 获取资源属性
  - 参数: resource_path (资源路径), property_name (属性名称)
  - 功能: 获取资源的指定属性值

### 6. 项目设置工具

#### 6.1 项目配置操作
- `set_project_setting`: 设置项目配置
  - 参数: setting_name (配置名称), setting_value (配置值)
  - 功能: 设置项目配置项的值
  
- `get_project_setting`: 获取项目配置
  - 参数: setting_name (配置名称)
  - 功能: 获取项目配置项的值

#### 6.2 输入映射操作
- `add_input_action`: 添加输入动作
  - 参数: action_name (动作名称), events (事件列表)
  - 功能: 添加新的输入动作
  
- `remove_input_action`: 删除输入动作
  - 参数: action_name (动作名称)
  - 功能: 删除指定的输入动作


## 实现优先级建议

### 第一优先级 (核心功能)
1. 文件系统操作工具 (create_file, write_to_file, delete_file)
2. 场景节点操作工具 (add_node, remove_node)
3. 脚本绑定操作工具 (attach_script, detach_script)
4. 信号连接操作工具 (connect_signal, disconnect_signal)

### 第二优先级 (增强功能)
1. 节点属性操作工具 (set_node_property, get_node_property)
2. 场景管理工具 (create_scene, save_scene)
3. 脚本内容操作工具 (add_script_function, modify_script_function)
4. 资源操作工具 (set_resource_property, get_resource_property)

### 第三优先级 (高级功能)
1. 目录操作工具
2. 节点高级操作工具 (duplicate_node, move_node)
3. 信号自定义操作工具
4. 项目设置工具
